# MVP仕様整理

## CORE VALUE：最適化された買い物リスト表示
献立、レシピ、在庫情報から計算して、各買い物1回に必要なリストを表示する

## 実現に必要な機能整理
### 在庫管理
- 鶏モモ肉やニンジン、醤油など、レシピの材料となる食材を、家庭単位で有無を登録・更新・削除
- MVP：「ある」「ない」の二択
- 今後：分量や個数は考慮した正確な在庫管理の実現

### レシピ管理
- 家庭内で共有するレシピを登録・更新・削除
- MVP:テンプレートレシピのみ用意し、後は家庭単位でレシピを登録してもらう
- MVP:レシピ登録時に材料テキストをペーストすると、Claude APIが食材を抽出してグローバルマスタと照合し、確認画面を経てDB保存する（食材のみ・分量はMVP以降）
  - マスタに存在しない食材は家庭専用食材として新規追加する
  - 家庭間でのレシピ共有は著作権上の問題があるため対象外
### 献立管理
- 家庭内で共有されたレシピを参照し、日付単位での献立を登録・更新・削除
- MVP:1日1レシピのみ
- MVP：献立完了操作により、そのレシピの食材一覧（デフォルト全チェック）を表示し、ユーザーが使い切った食材を選択して在庫「なし」に更新
- 今後：1日複数回のレシピ登録

### 買い物リスト表示(コア機能)
- 献立→レシピ→在庫を参照し、1回の買い物で購入すべき材料を計算し、表示する機能
- ユーザー操作(買い物完了後)により、買い物リストに表示される材料の一括在庫状況更新が可能
- MVP：家庭単位で買い物曜日を設定可能（デフォルト土曜）。頻度は週1回固定
- MVP：表示対象は常に直近の買い物サイクル（直近の買い物日〜次の買い物日の前日）
- MVP：対象期間に献立が未設定の場合はプレースホルダーを表示する
- MVP：買い物リストへの食材の手動追加は対象外。リストは献立・レシピ・在庫から自動計算されたもののみを表示する
- 今後：買い物頻度（週複数回など）を家庭単位で設定可能にし、それに応じた買い物リストの計算
- 今後：献立に含まれない食材の手動追加

### ユーザー認証/家庭(グループ)管理
- ユーザーがログインし、家庭単位でデータ（在庫、レシピ、献立、買い物リスト）を共有・管理
- 複数ユーザーが同じ家庭に所属し、データを共同編集可能
- MVP：Googleアカウント連携、家庭メンバーの手動登録
- 今後：招待機能（メール経由で家族を招待）、SNS連携ログイン

## 非機能要件

### パフォーマンス
- **買い物リスト表示**: 2秒以内に表示（コア機能のため最重要）
- **在庫更新**: 1秒以内に反映

### モバイル対応
- **レスポンシブデザイン**: スマホでの操作を最優先（買い物中に使用するため）
- **タッチ操作**: タップしやすいUI（ボタンサイズ、間隔）

### セキュリティ
- **認証**: Google OAuth 2.0による認証（パスワード管理不要）
- **セッション管理**: 適切なタイムアウト設定
- **アクセス制御**: 家庭単位でのデータ分離

### 可用性
- **エラーハンドリング**: ユーザーフレンドリーなエラーメッセージ
- **オフライン対応**: MVP では不要、今後検討
