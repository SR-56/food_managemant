# 食材管理アプリ UI設計書

## 設計方針

### コアバリュー
最適化された買い物リスト表示（献立、レシピ、在庫から計算）

### 非機能要件
- **モバイルファースト**: スマホでの操作を最優先
- **タップしやすさ**: ボタンサイズ最小44px、間隔12px以上
- **パフォーマンス**: 買い物リスト表示2秒以内、在庫更新1秒以内
- **シンプル**: MVPは最小限の機能に絞る

1. ログイン画面

5. レシピ管理画面

## 1. ログイン画面

### レイアウト
- 中央配置のログインカード
- 背景: シンプルなグラデーションまたは単色

### UI要素
- **ロゴ/アプリ名**: 上部中央
  - アプリ名: **FooCo**
  - アイコン: 食パン、茶碗ご飯、またはFooCoロゴ
- **キャッチコピー**:
  - アプリの説明文（1-2行）
  - 例: 「献立に基づいた買い物リストで、無駄のない食材管理」
- **Googleでログインボタン**:
  - Googleロゴ + 「Googleでログイン」テキスト
  - 幅100%
  - 高さ48px（タップしやすい）
  - 背景: 白（#ffffff）
  - ボーダー: グレー（#e5e7eb）
  - Googleのブランドガイドラインに準拠
- **エラーメッセージ表示エリア**:
  - ログインボタンの下
  - 赤い背景で目立たせる

### 認証フロー
1. 「Googleでログイン」ボタンをタップ
2. Googleの認証画面へリダイレクト
3. ユーザーがGoogleアカウントを選択・認証
4. アプリに戻り、ホーム画面へ遷移

### モバイル対応
- カード幅: スマホで90%、タブレット以上で最大400px

## 2. ホーム画面（ダッシュボード）

### レイアウト
- **ヘッダー**:
  - 左: FooCo ロゴ（食パンアイコン + FooCo テキスト）
  - 右: ユーザーメニュー（家庭名、ログアウト）
- **ナビゲーション**:
  - スマホ: 下部固定のボトムナビゲーション（5アイテム）
  - タブレット以上: 左サイドバー
- **メインエリア**:
  - 今週の買い物リストへのショートカット
  - 今週の献立サマリー
  - 在庫切れアラート

### ナビゲーションアイテム
1. **ホーム** (アイコン: Home)
2. **買い物リスト** (アイコン: ShoppingCart) ← コア機能
3. **献立** (アイコン: Calendar)
4. **レシピ** (アイコン: Book)
5. **在庫** (アイコン: Package)

### 今週の買い物リスト（カード）
- **ヘッダー**:
  - タイトル「今週の買い物リスト（土曜日）」
  - 日付表示
- **コンテンツ**:
  - 必要な食材の個数表示（例: 「8品目」）
  - 「リストを見る」ボタン → 買い物リスト画面へ遷移
- **スタイル**:
  - カードデザイン、シャドウ付き
  - primary colorでアクセント

### 今週の献立サマリー（カード）
- **ヘッダー**: 「今週の献立」
- **コンテンツ**:
  - 月〜日の7日分を簡易表示
  - 各日: 日付 + レシピ名（未設定なら「-」）
- **アクション**:
  - 「献立を編集」ボタン → 献立カレンダー画面へ遷移

### 在庫切れアラート（カード）
- **ヘッダー**: 「在庫切れの食材」
- **コンテンツ**:
  - 在庫が「ない」になっている食材のリスト（最大5件表示）
  - 「すべて見る」リンク → 在庫管理画面へ遷移
- **スタイル**:
  - 警告色（オレンジ系）でアクセント

## 3. 買い物リスト画面（コア機能）

### ヘッダー
- 左: 戻るボタン
- 中央: 「買い物リスト」
- 右: 日付表示（例: 2026/02/08 土曜日）

### メインエリア

#### 対象期間表示
- 「今週の買い物（2/8〜2/14）」

#### 買い物リスト
各食材をカード形式で表示:
- **食材名**: 大きめのフォント
- **必要な理由**:
  - 「〇〇（レシピ名）で使用」
  - 小さめのフォント、グレー
- **チェックボックス**:
  - 左側に配置
  - サイズ24px（タップしやすい）
  - チェックで「購入済み」マーク

#### リストのグループ化
- カテゴリ別に分類（野菜、肉類、調味料など）
- 各カテゴリにヘッダー

#### フッター固定ボタン
- **「買い物完了」ボタン**:
  - 幅100%
  - 高さ56px
  - primary color
  - タップで確認ダイアログ表示
  - 確認後、チェックした食材の在庫を一括更新（「ある」に変更）

### 確認ダイアログ
- タイトル: 「買い物を完了しますか？」
- 内容: 「チェックした食材の在庫を更新します」
- ボタン: 「キャンセル」「完了」

### パフォーマンス
- リスト表示: 2秒以内
- 仮想スクロール（リストが長い場合）

## 4. 献立カレンダー画面

### ヘッダー
- 左: 戻るボタン
- 中央: 「献立カレンダー」
- 右: 「+」ボタン（献立追加）

### 週間カレンダー
- 月〜日の7日分を縦スクロールで表示
- 各日のカード:
  - **日付ヘッダー**:
    - 曜日 + 日付（例: 「月 2/8」）
    - 土曜日は primary color でハイライト（買い物日）
  - **レシピ表示エリア**:
    - 登録済み: レシピ名を表示
    - 未登録: 「レシピを追加」ボタン（点線枠）
  - **アクション**:
    - タップでレシピ選択ダイアログ表示
    - 登録済みの場合、削除ボタン（×アイコン）も表示
    - 過去の日付かつ登録済みの場合、「献立完了」ボタンも表示

### 献立完了ダイアログ
- タイトル: 「使い切った食材を選んでください」
- コンテンツ:
  - そのレシピの食材一覧（チェックボックス、デフォルト全チェック）
  - 各食材: 食材名 + チェックボックス（左側）
- ボタン: 「キャンセル」「完了」
- 完了後: チェックした食材の在庫を「なし」に更新

### レシピ選択ダイアログ
- タイトル: 「レシピを選択」
- 検索バー: レシピ名で検索
- レシピリスト:
  - 各レシピをカード形式で表示
  - タップで選択、ダイアログを閉じる
- ボタン: 「キャンセル」

### 週の切り替え
- カレンダー上部に「< 先週 | 今週 | 来週 >」

## 5. レシピ管理画面

### レシピ一覧画面

#### ヘッダー
- 左: 戻るボタン
- 中央: 「レシピ」
- 右: 「+」ボタン（新規レシピ作成）

#### 検索バー
- placeholder: 「レシピ名で検索」
- 幅100%、高さ48px

#### レシピリスト
各レシピをカード形式で表示:
- **レシピ名**: 大きめのフォント
- **必要な食材**:
  - 食材名をカンマ区切りで表示（最大3つ、それ以上は「他〇品」）
  - 小さめのフォント、グレー
- **アクション**:
  - タップでレシピ詳細画面へ遷移

### レシピ詳細画面

#### ヘッダー
- 左: 戻るボタン
- 中央: レシピ名
- 右: 編集ボタン（鉛筆アイコン）

#### メインエリア
- **必要な食材**:
  - セクションヘッダー「必要な食材」
  - 食材リスト（各食材を1行で表示）
  - 各食材の右側に在庫ステータス表示:
    - 「在庫あり」（緑）
    - 「在庫なし」（赤）

#### フッターボタン
- 「レシピを削除」ボタン（削除色、下部固定）

### レシピ編集画面

#### ヘッダー
- 左: キャンセルボタン
- 中央: 「レシピ編集」（新規の場合「レシピ作成」）
- 右: 保存ボタン

#### フォーム
- **レシピ名**:
  - ラベル「レシピ名」
  - 入力欄、高さ48px
- **必要な食材**:
  - ラベル「必要な食材」
  - 食材選択エリア:
    - 追加済み食材のリスト（各食材に削除ボタン）
    - 「+ 食材を追加」ボタン
    - タップで食材選択ダイアログ表示

#### 食材選択ダイアログ
- タイトル: 「食材を選択」
- 検索バー: 食材名で検索
- 食材リスト:
  - 在庫管理の食材マスタから取得
  - チェックボックスで複数選択可能
- ボタン: 「キャンセル」「追加」

## 6. 在庫管理画面

### ヘッダー
- 左: 戻るボタン
- 中央: 「在庫管理」
- 右: フィルターボタン（アイコン）

### フィルターオプション（ドロップダウン）
- すべて
- 在庫あり
- 在庫なし

### 検索バー
- placeholder: 「食材名で検索」
- 幅100%、高さ48px

### 在庫リスト
各食材を1行で表示（リスト形式）:
- **左側**: 食材名
- **右側**: トグルスイッチ
  - ON（緑）: 在庫あり
  - OFF（グレー）: 在庫なし
  - タップで即座に切り替え、1秒以内に更新

### カテゴリ別グループ化
- 野菜、肉類、魚類、調味料など
- 各カテゴリにヘッダー

### 食材マスタについて（MVP）
- プリセットで基本的な食材リストを提供
- ユーザーが独自に追加する機能はMVP後に実装
- カテゴリは固定（野菜、肉類、魚類、調味料、その他）

## 7. 設定画面

### ヘッダー
- 左: 戻るボタン
- 中央: 「設定」

### メニューリスト

#### 家庭情報
- **家庭名**:
  - ラベル + 現在の家庭名表示
  - タップで編集ダイアログ
- **買い物日設定**:
  - ラベル「買い物日」
  - 現在の設定表示「毎週土曜日」（MVP固定）
  - 注記: 「※MVP版では変更できません」

#### 家庭メンバー
- セクションヘッダー「家庭メンバー」
- メンバーリスト:
  - 各メンバー: 名前 + メールアドレス
  - 自分には「（あなた）」表示
- 「+ メンバーを追加」ボタン
  - タップでメンバー追加ダイアログ

#### メンバー追加ダイアログ
- タイトル: 「メンバーを追加」
- 説明: 「Googleアカウントのメールアドレスを入力してください」
- フォーム:
  - メールアドレス入力欄
  - placeholder: "member@gmail.com"
- ボタン: 「キャンセル」「追加」
- 注記: 「※MVP版では手動で追加されます。招待メール機能は今後実装予定です」

#### 追加フロー（MVP）
1. メールアドレスを入力して「追加」
2. 家庭メンバーリストに追加される
3. 相手はそのGoogleアカウントでログインすると、自動的に同じ家庭に参加

#### アカウント
- **ログアウト**:
  - 赤いテキストボタン
  - タップで確認ダイアログ

## デザイントークン

### カラーパレット
- **Primary**:
  - メイン色（#C19A6B パンのような優しい茶色 - 温かみのある食卓をイメージ）
  - ボタン、アクセント、アクティブ状態
- **Secondary**:
  - サブ色（例: #f59e0b オレンジ系）
  - アラート、買い物日のハイライト
- **Background**:
  - #f9fafb（薄いグレー）
- **Surface**:
  - #ffffff（カード背景）
- **Error**:
  - #ef4444（赤）
- **Success**:
  - #10b981（緑）
- **Text Primary**:
  - #111827（濃いグレー）
- **Text Secondary**:
  - #6b7280（グレー）

### タイポグラフィ
- **見出し1**: 24px, bold
- **見出し2**: 20px, semi-bold
- **見出し3**: 16px, semi-bold
- **本文**: 14px, regular
- **キャプション**: 12px, regular

### スペーシング
- **余白**: 4px, 8px, 12px, 16px, 24px, 32px
- **カード間**: 16px
- **セクション間**: 24px

### コンポーネントサイズ
- **ボタン高さ**: 48px（プライマリ）、40px（セカンダリ）
- **入力欄高さ**: 48px
- **アイコンサイズ**: 24px
- **ボトムナビゲーション高さ**: 64px
- **ヘッダー高さ**: 56px

## アクセシビリティ

- すべてのタップ領域: 最小44×44px
- コントラスト比: WCAG AA準拠（最低4.5:1）
- フォーカス状態の明確な表示
- スクリーンリーダー対応（aria-label）
